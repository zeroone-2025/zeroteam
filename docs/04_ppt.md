## 1. Pain Point (30초)

> 청중이 공감할 수 있는 문제를 제시하세요.

**"여러분, 듣고 싶은 노래를 틀었는데 전혀 다른 노래가 나온 적 있으신가요?"**

- 문제 상황 설명: 내가 원하는 노래를 선택했는데, 실제로는 다른 노래가 나오거나, 느릿느릿 0.5배속으로 재생되거나, 심지어 아무 소리도 안 들리는 상황
- 현재 해결 방법의 한계: 기존 음악 스트리밍은 너무 친절하게 원하는 걸 다 줘버림. 재미가 없어요.
- 임팩트: 그래서 저희가 물었습니다. "음악 앱이 꼭 사용자 말을 들어야 하나요?"

> **AI 프롬프트** -- 공감 가는 도입부 스크립트 작성
>
> ```
> @docs/01_plan.md 참고해서 발표 도입부(30초) 스크립트를 작성해줘.
> 청중이 공감할 질문으로 시작하고, 구체적 상황 묘사 후
> "그래서 저희가 만들었습니다"로 자연스럽게 연결해줘.
> 말하듯이 자연스럽게, 200자 이내로.
> ```
>
> **TIP**: 도입부에서 청중의 공감을 얻으면 나머지 발표가 훨씬 수월해집니다. 질문은 손을 들게 하는 형식이 효과적입니다.

## 2. Solution (30초)

> 우리 서비스가 이 문제를 어떻게 해결하는지 명확하게 설명하세요.

**"그래서 저희는 I'M KING을 만들었습니다."**

- 한줄 소개: 원하는 노래를 선택하면 다른 노래, 0.5배속, 무음 중 하나로 배신당하는 서비스입니다.
- 핵심 가치: 배신당하는 순간의 황당함과 웃음. 사용자가 주인인 척하지만, 왕은 저희입니다.
- 차별점: 기존 음악 앱은 모두 "사용자 경험 최적화"를 외칩니다. 원하는 걸 빠르게, 끊김 없이. 반면 I'M KING은 정반대입니다. 의도적으로 기대를 배신하고, 그 배신이 콘텐츠가 됩니다. 경쟁 서비스가 없는 이유는 아무도 이걸 만들 생각을 안 했기 때문입니다. 저희가 처음입니다.

> **AI 프롬프트** -- 솔루션 소개 스크립트 작성
>
> ```
> @docs/01_plan.md @docs/03_marketing.md 참고해서 발표 Solution 파트(30초) 스크립트를 작성해줘.
> 서비스 이름과 한줄 정의, 핵심 가치, 차별점 1가지를 포함하고,
> 마케팅 문서의 슬로건과 엘리베이터 피치를 활용해서 200자 이내로.
> ```
>
> **TIP**: 여기서 03_marketing.md의 엘리베이터 피치를 다듬어 사용하면 일관된 메시지를 전달할 수 있습니다.

## 3. Features - 데모 (2분)

> 실제 화면을 보여주며 핵심 기능을 시연하세요.

### 기능 1: 노래 검색 & 선택 UI
- 시연 순서: 검색창에 원하는 곡 입력 → 결과 목록에서 곡 선택 → "재생" 버튼 클릭
- 강조 포인트: 완전히 평범한 음악 앱처럼 보입니다. 사용자는 아무것도 모릅니다.

### 기능 2: 랜덤 배신 엔진
- 시연 순서: 재생 버튼 클릭 → TYPE A / B / C 중 하나 랜덤 발동 → 결과 확인
  - TYPE A: 전혀 다른 노래 재생
  - TYPE B: 선택한 곡이 0.5배속으로 재생
  - TYPE C: 재생 중 UI는 정상 작동하지만 무음
- 강조 포인트: "어? 이게 뭐야?" 반응이 나오는 순간이 핵심 데모 포인트. 청중에게도 직접 눌러보게 하면 효과적.

> **AI 프롬프트** -- 데모 시나리오 작성
>
> ```
> @docs/01_plan.md @docs/02_dev.md 참고해서 2분짜리 데모 시나리오를 작성해줘.
> 시작 화면, 기능별 시연 순서(클릭/입력 위치, 발표자 대사, 주목 포인트),
> 기능 간 전환 멘트, 마무리를 포함해줘.
> 발표자가 보면서 따라할 수 있도록 단계별로 작성해줘.
> ```
>
> **TIP (사전 준비)**: 데모 전에 반드시 준비하세요:
> - 테스트 데이터를 미리 입력해 둘 것 (빈 화면은 감점 요소)
> - 네트워크 끊김 대비 백업 스크린샷/영상
> - 브라우저 즐겨찾기에 데모 시작 URL 등록
> - 불필요한 브라우저 탭, 알림 모두 닫기

## 4. Scalability (1분)

> 이 프로젝트의 확장 가능성과 비전을 제시하세요.

- **기술적 확장**: 배신 유형을 지속적으로 추가해 예측 불가능성 강화. 영상, 이미지 등 음악 외 콘텐츠로 확장 가능.
- **비즈니스 확장**: 배신 결과 화면 광고 삽입으로 수익화 / 유료 구독 시 더 창의적인 프리미엄 배신 유형 제공 / 수능·크리스마스 등 시즌 이벤트 에디션 운영
- **임팩트**: 국내 밈·숏폼 콘텐츠를 소비하는 2030 약 1,200만 명이 잠재 사용자입니다. I'M KING은 단순한 장난 앱이 아니라, 배신이 콘텐츠가 되는 새로운 포맷을 제시합니다.

**"당신이 주인인 줄 알았겠지만 — I'M KING."**

> **AI 프롬프트** -- 확장 비전 스크립트 작성
>
> ```
> @docs/01_plan.md 참고해서 발표 마무리(1분) 스크립트를 작성해줘.
> 기술적 확장 계획, 비즈니스 확장(수익화/시장 확대), 기대 임팩트(숫자 포함),
> 인상 깊은 클로징 한마디를 포함해줘.
> 현실적이면서도 꿈이 있는 톤으로, 300자 이내로.
> ```
>
> **TIP**: "감사합니다"로 끝내지 말고, 서비스의 비전을 담은 한마디로 마무리하면 기억에 남습니다.

**"감사합니다. 질문 받겠습니다."**
